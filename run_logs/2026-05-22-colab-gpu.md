The GitHub version used for this run still exposed the package as `gomoku_ai`.

## Training Commands

Small smoke run:

```bash
PYTHONPATH=src python -m gomoku_ai.train \
  --iterations 1 \
  --games 5 \
  --epochs 1 \
  --board-size 9 \
  --win-length 5 \
  --mcts-sims 8 \
  --checkpoint /content/drive/MyDrive/gomoku_checkpoints/9x9.pt
```

Follow-up run:

```bash
PYTHONPATH=src python -m gomoku_ai.train \
  --iterations 3 \
  --games 20 \
  --epochs 1 \
  --board-size 9 \
  --win-length 5 \
  --mcts-sims 16 \
  --allow-immediate-loss \
  --checkpoint /content/drive/MyDrive/gomoku_checkpoints/9x9.pt
```

`--allow-immediate-loss` keeps self-play training faster by disabling the expensive terminal safety checks during training. Human play can still use the safety checks.

## Checkpoint Workflow

Check the saved checkpoint in Drive:

```bash
ls -lh /content/drive/MyDrive/gomoku_checkpoints/9x9.pt
```

Download it from Colab:

```python
from google.colab import files
files.download("/content/drive/MyDrive/gomoku_checkpoints/9x9.pt")
```

Move the downloaded checkpoint into the local repo before human testing:

```bash
mv ~/Downloads/9x9.pt checkpoints/9x9.pt
```

The checkpoint file stays out of Git because `checkpoints/*.pt` is ignored.

## Local Human Test

The local package has been renamed to `gomoku_agent`:

```bash
PYTHONPATH=src python -m gomoku_agent.play \
  --checkpoint checkpoints/9x9.pt \
  --board-size 9 \
  --win-length 5 \
  --mcts-sims 64
```

## Notes

The 9x9 workflow is now usable end to end: train on Colab GPU, save checkpoints in Google Drive, download a checkpoint, and test it locally against a human player.
